# Milestone v1.0: MVP

**Status:** ✅ SHIPPED 2026-05-30
**Phases:** 1–5
**Total Plans:** 17

## Overview

CipherBench v1.0 is a Python SDK and CLI benchmark that tests LLMs on a stateful, cross-character, feedback-hidden cipher challenge. Built from the inside out: the rule engine establishes the information boundary first, then puzzle generation layers reproducibility on top, then session infrastructure makes the benchmark runnable end-to-end with real API calls, then scoring turns raw sessions into AGI proximity signals, and finally the session inspector closes the CLI surface.

All 20 v1 requirements delivered and validated. 149 tests passing at close.

## Milestone Summary

**Key Decisions:**
- All three cipher layers (state, cross-char, hidden feedback) in v1 — each targets a distinct LLM shortcut; combining them is the benchmark's core thesis
- Layered pure-function architecture in `engine/layers.py` — composable, independently testable, no shared state
- Procedural generation over fixed puzzle set — prevents memorization and dataset contamination
- Provider-agnostic LiteLLMAdapter — routes to Anthropic, OpenAI, Google, and 100+ others via single `complete()` call
- 3-module scoring split (scorer/reporter/report_writer) — clean separation of computation, display, and I/O
- Option B symmetric encoding for ground_truth — random seeded GT + symmetric state+cross_char encoding; submitting GT always scores is_correct=True

**Issues Resolved:**
- CR-01: Cross-char layer was a no-op when GT=alphabet[0]*n ("AAAAA") — fixed with Option B symmetric encoding
- CR-02: Session resume did not restore valid_attempts — fixed with valid_attempts_start parameter
- CR-03: LiteLLMAdapter returned None on empty content — fixed with None guard
- CR-04: Session resume did not check difficulty — fixed with difficulty_tier match
- WR-01 through WR-06: UTF-8 encoding, glob.escape, atomic write, api_base rename, PROBE: prefix strip, max_attempts param
- Post-review WR-01..WR-04: total_iterations resume, output_length in extractor, # placeholder in prompt, RuleEngine attempt limit enforced

**Technical Debt Deferred:**
- `inspect_session` raises SystemExit(1) for CLI use case — library callers should wrap in try/except SystemExit (flagged in Phase 05 decision log)
- v2 requirements (INTEG-01..03, EXT-01..03) scoped to future milestone
