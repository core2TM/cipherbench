## v1 Requirements — All Complete

### Rule Engine
- [x] **RULE-01**: State layer — same probe in different round produces different output ✓ Phase 1
- [x] **RULE-02**: Cross-character layer — shift on one output depends on a different input character ✓ Phase 1
- [x] **RULE-03**: Hidden feedback — only correctness score returned, never the encoded output ✓ Phase 1
- [x] **RULE-04**: Information boundary — `score_attempt()` is the only public interface ✓ Phase 1

### Puzzle Generation
- [x] **GEN-01**: Reproducible puzzle from integer seed ✓ Phase 2
- [x] **GEN-02**: SHA-256 hash verification — same seed asserts same hash ✓ Phase 2
- [x] **GEN-03**: Configurable difficulty parameters (alphabet range, state_change_rate, cross_char_depth) ✓ Phase 2
- [x] **GEN-04**: Explicit `rng` parameter threading — no global `random.seed()` ✓ Phase 1

### Session & CLI
- [x] **SESS-01**: `cipherbench run` feeds puzzles to model via LiteLLM, records JSON sessions ✓ Phase 3
- [x] **SESS-02**: `cipherbench play` presents puzzles to human, records same JSON schema ✓ Phase 3
- [x] **SESS-03**: `cipherbench inspect` replays stored session trace ✓ Phase 5
- [x] **SESS-04**: 50-run sequential determinism test passes ✓ Phase 3

### Model Adapters
- [x] **ADAPT-01**: Single `complete(messages) -> str` routes to any LiteLLM provider ✓ Phase 3
- [x] **ADAPT-02**: Token budget check at session init — warns and aborts on overflow ✓ Phase 3
- [x] **ADAPT-03**: Rate-limit handling with exponential backoff + per-attempt checkpoint ✓ Phase 3
- [x] **ADAPT-04**: Regex + fallback parsing of probe from freeform model response ✓ Phase 3

### Scoring & Reporting
- [x] **SCORE-01**: Success rate across sessions ✓ Phase 4
- [x] **SCORE-02**: Efficiency score per session ✓ Phase 4
- [x] **SCORE-03**: AGI proximity score normalized by human baseline ✓ Phase 4
- [x] **SCORE-04**: Per-difficulty-tier breakdown ✓ Phase 4
