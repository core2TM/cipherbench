    - __init__(self, base_shifts: list[int], k: int, difficulty: DifficultyConfig, ground_truth: str) -> None
      Private attrs: self._base_shifts = base_shifts, self._k = k, self._alphabet = difficulty.alphabet, self._ground_truth = ground_truth, self._round = 1

      Increment self._round += 1
      Compute current_target = self._encode_for_round(round_num)
      Return AttemptScore(score=count_correct(guess, current_target), max_score=len(guess), is_correct=(count_correct(guess, current_target) == len(guess)))
    - _encode_for_round(self, round_num: int) -> str  (private helper)
      Call apply_state_layer(self._ground_truth, self._base_shifts, round_num, self._alphabet) → shifted_indices
      Call apply_cross_char_layer(shifted_indices, self._ground_truth, self._k, self._alphabet) → encoded str
      Return encoded str
    - NO other methods. No reset(), no get_key(), no cipher_text property, no __repr__ that leaks state.

      n = difficulty.output_length  (D-06: fixed 5)

      Return RuleEngine(base_shifts=base_shifts, k=k, difficulty=difficulty, ground_truth=ground_truth)
    - ground_truth = "A" * n is intentionally fixed: same seed → same ground_truth → same shifts → same sequence; the model's task is to submit the ciphertext that scores max_score for the current round, i.e. submit _encode_for_round(current_round) for whichever round they are in

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| External caller → RuleEngine.score_attempt | Only interaction point. Untrusted guess string enters here. |
| RuleEngine private state → public return value | AttemptScore must carry no cipher state. |
| Factory (create_rule_engine) → rng | rng is isolated to the factory call; must not bleed to global random state |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Information Disclosure | RuleEngine private attributes via Python introspection | accept | Single-underscore convention does not prevent access via engine._base_shifts; documented in class docstring as accepted risk for a research tool; boundary enforcement via test suite (test_no_public_key_accessor); no mechanism prevents deliberate introspection by the Python process that created the engine, which is the same process — acceptable for a local research tool with no privilege separation |
| T-03-02 | Information Disclosure | Exception messages leaking cipher state | mitigate | ValueError messages in score_attempt use generic language ("guess length X does not match output_length Y", "guess contains characters outside the configured alphabet") — no _base_shifts, _k, or _ground_truth values in message strings; verified by test_score_attempt_rejects_wrong_length checking the exception message does not contain "AAAAA" or shift values |
| T-03-03 | Tampering | State bleed via reused RuleEngine instance | mitigate | create_rule_engine factory enforces fresh-instance-per-session (D-10); test_fifty_sequential_runs_are_deterministic catches any state bleed; conftest fixtures are function-scoped so each test gets an isolated instance |
| T-03-04 | Information Disclosure | Global random state inference | mitigate | create_rule_engine uses random.Random(seed) isolated instance (D-11); test_rng_does_not_pollute_global_random verifies global random.getstate() is unchanged after factory call; adversary monitoring global random.random() cannot infer the puzzle's RNG sequence |
| T-03-05 | Tampering | Adversarial guess triggering exception path that reveals round count | mitigate | score_attempt increments _round only after validation passes — a rejected guess does not advance the round counter; ValueError raised before any state mutation |
| T-03-SC | Tampering | npm/pip/cargo installs | mitigate | No new packages in Plan 03; all dev dependencies installed and audited in Plan 01 |
</threat_model>
