# Walking Skeleton: CipherBench Phase 1

**Phase:** 1 — Rule Engine
**Created:** 2026-05-28
**Approach:** Functional-core / OOP-shell with locked information boundary

---

## What the Skeleton Delivers

The thinnest possible working end-to-end slice of the rule engine:

- A caller can `import cipherbench` and call `create_rule_engine(seed=42)` to obtain a `RuleEngine` instance
- The caller can call `engine.score_attempt("ABCDE")` and receive an `AttemptScore(score, max_score, is_correct)`
- The cipher key and internal state are unreachable from outside the `RuleEngine` boundary
- Every subsequent phase imports from `cipherbench` directly — this is the stable API surface

## Architectural Decisions

### Framework and Runtime

| Decision | Choice | Rationale |
|----------|--------|-----------|
| Language | Python 3.11+ | Project target; type annotation syntax requires 3.11 |
| Packaging | pyproject.toml + hatchling | PEP 517/518 compliant; zero-config src layout |
| Dependency management | uv (pip fallback) | Lockfile support; 10-100x faster installs |
| Testing | pytest 8.x + hypothesis 6.x | Fixture injection maps to cipher state init; property-based tests for combinatorial edge cases |
| Runtime dependencies | stdlib only (Phase 1) | random.Random, dataclasses — no installs needed for production code |

### Directory Layout

```
cipherbench/              # installable package
    __init__.py           # re-exports public API: AttemptScore, DifficultyConfig, RuleEngine, create_rule_engine
    types.py              # frozen dataclasses: AttemptScore, DifficultyConfig
    engine/
        __init__.py
        layers.py         # pure functions: apply_state_layer, apply_cross_char_layer, count_correct
        rule_engine.py    # RuleEngine class + create_rule_engine factory

tests/
    conftest.py           # shared fixtures: default_difficulty, rule_engine_seed_42, rule_engine_seed_0
    unit/
        __init__.py
        test_engine/
            __init__.py
            test_types.py       # AttemptScore, DifficultyConfig invariants
            test_layers.py      # pure layer function unit tests
            test_rule_engine.py # RuleEngine integration tests + boundary enforcement
            test_seeding.py     # GEN-04: RNG isolation, 50-run determinism
    test_properties.py    # Hypothesis property-based tests

pyproject.toml            # build config, pytest config, dev dependencies
uv.lock                   # pinned lockfile (generated by uv)
```

### Core API Surface (stable contract for downstream phases)

```
cipherbench.create_rule_engine(seed: int, difficulty: DifficultyConfig = DifficultyConfig()) -> RuleEngine
cipherbench.RuleEngine.score_attempt(guess: str) -> AttemptScore
cipherbench.AttemptScore.score: int            # 0..output_length
cipherbench.AttemptScore.max_score: int        # output_length (5)
cipherbench.AttemptScore.is_correct: bool      # score == max_score
cipherbench.DifficultyConfig.alphabet: str     # default "ABCDEFGHIJKLMNOPQRSTUVWXYZ"
cipherbench.DifficultyConfig.output_length: int  # default 5
```

**Every downstream phase imports from `cipherbench` top-level only.** Internal module paths (`cipherbench.engine.rule_engine`) are implementation detail and may change.

### Key Architectural Choices (non-negotiable, locked by CONTEXT.md)

| Choice | Lock | What It Means |
|--------|------|---------------|
| Single public method on RuleEngine | D-09 | `score_attempt` is the only non-private method — ever |
| Factory pattern for construction | D-10 | `create_rule_engine(seed, difficulty)` is the only authorized constructor |
| Explicit RNG threading | D-11 | `rng = random.Random(seed)` constructed once in factory; passed to every sub-function |
| Aggregate score only | D-01/D-02 | AttemptScore has `score: int`, `max_score: int`, `is_correct: bool` — no per-position breakdown |
| Fixed 5-char output | D-06 | `output_length = 5` in DifficultyConfig default — consistent across all puzzles |
| Round-multiplier state evolution | D-07 | `effective_shift = base_shift * round_num` (linear); D-08: multiplier applies to shifts only, not to cross-char `k` |
| Index-based cross-char mixing | D-04 | Pull model: output[j] += alphabet.index(input[(j-k) % N]); k is puzzle-level param |

### Data and Storage

- No storage in Phase 1 — pure in-memory computation
- Session persistence is Phase 3 (JSON files)
- No SQLite until Phase 3

### Auth and Security

- No user auth (no users in Phase 1)
- Information boundary: cipher key and ground-truth are private attributes (`_` prefix); public interface is `score_attempt` only
- Input validation on every `score_attempt` call: length check + alphabet membership check
