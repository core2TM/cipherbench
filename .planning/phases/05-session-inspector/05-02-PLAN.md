## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CLI args → inspect_command | --sessions-dir and session_id are user-supplied strings |
| sessions/*.json → inspector.py | JSON files on disk may be malformed, crafted, or missing |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01 | Tampering | `--sessions-dir` in inspect_command | mitigate | `Path(sessions_dir).resolve()` applied in inspect_command before passing to inspector — identical discipline to score_command (T-04-06). Comment: `# ASVS V5: resolve prevents path traversal (T-04-06)` |
| T-05-02 | Denial of Service | malformed/corrupted session JSON | mitigate | `try/except (json.JSONDecodeError, OSError)` around json.load() in inspect_session — graceful error message + SystemExit(1), same pattern as scorer.py |
| T-05-03 | Spoofing | filename stems containing shell metacharacters | accept | Stems displayed as strings through Rich console only; never passed to shell or eval; no injection surface |
| T-05-04 | Information Disclosure | inspector reads all session JSON files for stem listing | accept | Local single-user CLI; all JSON files are written by the same user/process via cipherbench run/play; no confidentiality boundary crosses |
| T-05-SC | Tampering | npm/pip/cargo installs | accept | No new packages — all packages slopcheck-verified [OK] in RESEARCH.md Package Legitimacy Audit; no install task in this plan |
