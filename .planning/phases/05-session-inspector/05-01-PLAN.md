<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CLI args → inspector.py | --sessions-dir is user-supplied; session_id arg is user-supplied |
| sessions/*.json → inspector.py | JSON files on disk may be malformed or crafted |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01 | Tampering | `--sessions-dir` in inspect_command | mitigate | `Path(sessions_dir).resolve()` in inspect_command before passing to inspector — same discipline as score_command (T-04-06). Applied in Plan 02 app.py modification. |
| T-05-02 | Denial of Service | malformed/corrupted session JSON | mitigate | `try/except (json.JSONDecodeError, OSError)` around json.load() — graceful error message + SystemExit(1), same as scorer.py pattern |
| T-05-03 | Spoofing | filename stems containing shell metacharacters | accept | Stems are only displayed as strings via Rich console, never passed to shell; no injection risk |
| T-05-SC | Tampering | npm/pip/cargo installs | accept | No new packages in this phase — all packages already installed and slopcheck-verified (see RESEARCH.md Package Legitimacy Audit) |
</threat_model>
