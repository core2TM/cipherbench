      def _compute_hash(base_shifts: list, k_list: list, ground_truth: str) -> str
      Serialization: json.dumps({"base_shifts": base_shifts, "ground_truth": ground_truth, "k_list": k_list}, sort_keys=True).encode()
      Note: k_list is ALWAYS serialized as a JSON array, never unwrapped to scalar (Pitfall 4 from RESEARCH.md).
      Returns hashlib.sha256(payload).hexdigest()

    GENERATE_PUZZLE FACTORY (GEN-01, GEN-02, D-06):
      def generate_puzzle(seed: int, difficulty: DifficultyConfig = None) -> Puzzle
      If difficulty is None, set difficulty = DifficultyConfig()
      Call engine = create_rule_engine(seed, difficulty)
      Call puzzle_hash = _compute_hash(engine._base_shifts, engine._k_list, engine._ground_truth)
      Return Puzzle(seed=seed, difficulty=difficulty, puzzle_hash=puzzle_hash)
      This is trusted factory access — accessing engine._base_shifts, engine._k_list, engine._ground_truth is intentional per PATTERNS.md.

    VERIFY_PUZZLE (GEN-02, D-09):
      def verify_puzzle(puzzle: Puzzle) -> None
      Re-runs: engine = create_rule_engine(puzzle.seed, puzzle.difficulty)
      Re-derives: expected = _compute_hash(engine._base_shifts, engine._k_list, engine._ground_truth)
      If expected != puzzle.puzzle_hash: raise ValueError(f"hash mismatch: expected {expected}, got {puzzle.puzzle_hash}")

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| External caller → generate_puzzle(seed, difficulty) | seed and difficulty are caller-supplied; generate_puzzle validates difficulty via DifficultyConfig.__post_init__ (Wave 1) |
| generate_puzzle → engine private state | Trusted factory access — generate_puzzle reads engine._base_shifts, engine._k_list, engine._ground_truth directly; this is intentional and equivalent to a friend function |
| verify_puzzle error message → caller | Error message includes hash hex strings only (not cipher key or base_shifts values) — consistent with RULE-04 information boundary philosophy |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-03-A | Spoofing | SHA-256 hash stored in Puzzle.puzzle_hash | accept | SHA-256 collision resistance is negligible risk for puzzle identifiers (2^128 collision resistance); hash serves as reproducibility proof, not a security-critical secret |
| T-02-03-B | Tampering | Crafted Puzzle(seed=X, difficulty=Y, puzzle_hash=Z) bypassing verify_puzzle | mitigate | verify_puzzle re-derives hash from seed+difficulty every time; there is no path to construct a valid puzzle_hash without running the full RNG derivation |
| T-02-03-C | Information Disclosure | verify_puzzle error message exposing hash | accept | Hash hex strings (64-char hex) do not expose cipher key, base_shifts, k_list, or ground_truth values; RULE-04 information boundary preserved |
| T-02-03-D | Denial of Service | generate_puzzle with arbitrary seed triggering large computation | accept | All RNG operations are O(output_length) = O(5); no unbounded computation path exists |
| T-02-03-E | Tampering | json.dumps hash serialization cross-platform stability | accept | json.dumps with sort_keys=True on int/str values is stable across Python versions and platforms per RESEARCH.md Pattern 7 verification |
| T-02-SC | Tampering | npm/pip/cargo installs | accept | No new packages — stdlib only (hashlib, json, dataclasses, all built-in) |
</threat_model>
