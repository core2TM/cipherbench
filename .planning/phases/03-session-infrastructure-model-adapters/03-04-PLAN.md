    - `run(self) -> dict`:
      1. `alphabet = self._puzzle.difficulty.alphabet`; `output_length = self._puzzle.difficulty.output_length`; `max_score = output_length`

      3. `valid_attempts = 0`
      4. While `valid_attempts < MAX_ATTEMPTS`:
         a. `_show_attempt_history(self._session_record["attempts"], max_score)` (empty table on first attempt)

         c. `attempt_score = self._engine.score_attempt(raw)`
         d. Build attempt entry: `probe=raw`, `score=attempt_score.score`, `max_score=max_score`, `is_correct=attempt_score.is_correct`, `raw_response=None`, `extraction_failed=False`, `attempt_num=len(...)+1`
         e. Append to attempts list; call `self._writer.write_checkpoint(self._session_record)`
         f. `_show_score_line(attempt_score.score, max_score, attempt_score.is_correct)`
         g. Increment `valid_attempts`; if `attempt_score.is_correct`: break
      5. Final answer: `raw_ans = typer.prompt("Submit final answer (ANSWER: XXXXX)").strip().upper()` — extract 5-char sequence from input (strip leading "ANSWER:" if present). Validate length and alphabet; if invalid, re-prompt once.
      6. `outcome = "success" if any(a["is_correct"] for a in self._session_record["attempts"]) else "failure"`

      8. Return session record.

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-04-01 | Tampering | typer.prompt() input | mitigate | _validate_probe() checks length == output_length and all chars in alphabet before passing to engine.score_attempt(); invalid input causes re-prompt not engine call |
| T-03-04-02 | Information Disclosure | run_command — model string in session JSON | mitigate | Only the model string (e.g., "anthropic/claude-opus-4-7") is stored; no API key values are stored or logged; LiteLLM reads keys from env vars (D-14) |
| T-03-04-03 | Tampering | --output-dir path | accept | Single-user researcher CLI; path traversal risk documented and accepted for v1 (per RESEARCH.md security threat table); --output-dir defaults to ./sessions |
| T-03-04-04 | Tampering | --litellm-config path | accept | Escape hatch for proxy config; user-controlled; not security-sensitive in v1 single-user context |
</threat_model>
